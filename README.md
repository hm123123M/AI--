# AI智能体开发教学项目

这是一个基于Python的AI智能体开发教学项目，旨在帮助学习者掌握AI智能体开发的核心概念和实践技能。

## 项目结构

```
AI课程/
├── venv/                    # Python虚拟环境
├── practice01/              # 实践练习目录1：基础LLM调用
│   └── llm_client.py        # LLM客户端测试脚本
├── practice02/              # 实践练习目录2：交互式聊天
│   └── chat_client.py       # 终端聊天客户端（支持流式输出）
├── practice03/              # 实践练习目录3：聊天历史压缩
│   └── chat_client.py       # 带聊天历史压缩功能的终端聊天客户端
├── practice04/              # 实践练习目录4：5W信息提取与历史搜索
│   └── chat_client.py       # 带5W提取和聊天历史搜索功能的客户端
├── .env                     # 环境变量配置文件
├── .env.example             # 环境变量配置模板
├── .gitignore               # Git忽略文件配置
├── requirements.txt         # 项目依赖列表
├── log.txt                  # 5W关键信息日志文件（自动生成）
└── README.md               # 项目说明文档
```

## 文件功能说明

### 1. practice01/llm_client.py

**功能用途：**
- 读取项目根目录的.env文件内容
- 使用Python标准http.client库访问用户定义的LLM服务
- 统计token消耗、请求时间和token速度
- 解析和展示LLM响应结果

**教学目标：**
- 学习如何读取和解析.env环境配置文件
- 掌握Python标准库http.client的使用方法
- 理解OpenAI兼容协议的LLM API调用方式
- 学习如何统计API调用的性能指标（token消耗、时间、速度）
- 掌握JSON数据的序列化和反序列化
- 学习错误处理和调试技巧

### 2. practice02/chat_client.py

**功能用途：**
- 终端界面交互式聊天，支持用户输入消息
- 支持工具调用功能，实现文件操作和网络访问能力
- 自动维护历史聊天记录，并添加到上下文
- 持续循环聊天，直到用户按Ctrl+C退出

**支持的工具：**
1. **list_files** - 列出目录下的文件及其属性（大小、创建时间、修改时间）
2. **rename_file** - 重命名指定目录下的文件
3. **delete_file** - 删除指定目录下的文件
4. **create_file** - 在指定目录下创建新文件并写入内容
5. **read_file** - 读取指定目录下文件的内容
6. **curl_request** - 通过HTTP请求访问网页并返回内容（支持GET/POST方法）
7. **get_datetime** - 获取当前日期时间（自动附加到每次请求中）
8. **get_weather** - 获取指定城市的天气预报（支持今天/明天/后天）

**教学目标：**
- 学习如何实现工具调用功能
- 掌握系统提示词的设计方法
- 理解工具调用的解析和执行流程
- 学习如何将工具执行结果反馈给LLM
- 理解多轮对话中工具调用的完整流程
- 掌握文件操作和网络请求的安全注意事项
- 学习HTTP客户端编程和网页内容获取

### 3. practice03/chat_client.py

**功能用途：**
- 在practice02基础上增加聊天历史压缩功能
- 当聊天历史超过5轮或上下文长度超过3000字符时，自动触发LLM总结
- 对前70%的对话内容进行压缩总结，保留最后30%的原文内容
- 自动计算压缩率并显示压缩结果
- 持续显示当前历史轮数和上下文长度
- **5W关键信息提取** - 每5轮聊天自动提取关键信息（Who、What、When、Where、Why）并保存到日志文件
- **聊天历史搜索** - 支持通过/search命令或工具调用来查找聊天历史记录

**核心功能：**
1. **聊天历史检测** - 自动检测聊天轮数和上下文长度
2. **智能压缩策略** - 前70%内容总结压缩，后30%保留原文
3. **压缩效果展示** - 显示原长度、新长度和压缩率
4. **自动触发机制** - 满足条件时自动执行压缩，无需用户干预
5. **5W信息提取** - 每5轮聊天主动触发LLM提取关键信息
6. **日志持久化** - 自动将5W信息增量保存到 `D:\人工智能提示词\AI课程\log.txt`
7. **历史搜索** - 支持 `/search` 命令和 `search_chat_history` 工具调用

**新增工具：**
- **search_chat_history** - 读取本地聊天历史记录文件，用于查找之前的对话内容

**教学目标：**
- 学习如何管理和优化长对话历史
- 理解上下文窗口限制的解决方案
- 掌握对话总结的实现方法
- 学习如何平衡压缩率和信息保留
- 理解自动触发机制的设计思路
- 掌握性能监控指标的添加方法
- 学习如何从对话中提取关键信息（5W规则）
- 理解日志持久化和增量更新的实现
- 掌握工具调用的扩展方法

### 4. practice04/chat_client.py

**功能用途：**
- 在practice03基础上完善5W关键信息提取和聊天历史搜索功能
- **5W关键信息提取** - 每5轮聊天自动提取关键信息（Who、What、When、Where、Why）
- **日志增量更新** - 将提取的5W信息增量保存到 `D:\人工智能提示词\AI课程\log.txt`
- **聊天历史搜索** - 支持通过`/search`命令或`search_chat_history`工具调用查找历史记录
- 当用户表达"查找聊天历史"意图时，自动结合历史记录完成完整的LLM请求

**核心功能：**
1. **5W信息提取** - 每5轮聊天主动触发LLM提取关键信息
   - Who（谁）：参与对话的人物或主体
   - What（做了什么）：对话中涉及的主要事件或行为
   - When（什么时候）：涉及的时间（可选）
   - Where（在何处）：涉及的地点（可选）
   - Why（为什么）：做这件事的原因或目的（可选）
2. **日志增量更新** - 自动将5W信息追加保存到日志文件，支持多次提取累积
3. **聊天历史搜索** - 支持多种触发方式：
   - `/search`命令前缀触发
   - 用户表达"查找聊天历史"意图时触发
   - LLM通过工具调用触发
4. **智能历史结合** - 将日志内容与用户请求结合，完成完整的LLM请求

**新增工具：**
- **search_chat_history** - 读取本地聊天历史记录文件，用于查找之前的对话内容

**教学目标：**
- 学习如何从对话中提取关键信息（5W规则）
- 理解日志持久化和增量更新的实现方法
- 掌握工具调用的扩展和触发机制
- 学习如何结合历史记录完成智能问答
- 理解意图识别和命令触发的实现

## 环境配置

### 1. 安装依赖

```bash
# 使用虚拟环境中的pip安装依赖
venv\Scripts\pip.exe install -r requirements.txt
```

### 2. 配置环境变量

复制 `.env.example` 文件为 `.env`，并填写正确的LLM配置参数：

```env
# API基础URL
BASE_URL=http://127.0.0.1:1234/v1

# 模型名称
MODEL=qwen/qwen3.5-4b

# API密钥
API_KEY=your_api_key_here

# 温度参数（可选）
TEMPERATURE=1

# 最大生成长度（可选）
MAX_TOKENS=4096
```

### 3. 运行测试

```bash
# 运行LLM客户端测试
python practice01/llm_client.py

# 运行终端聊天客户端
python practice02/chat_client.py

# 运行带聊天历史压缩功能的终端聊天客户端
python practice03/chat_client.py

# 运行带5W信息提取和聊天历史搜索功能的终端聊天客户端
python practice04/chat_client.py
```

## 技术要点

1. **环境变量加载**：使用自定义函数读取.env文件，无需依赖第三方库
2. **URL解析**：支持HTTP/HTTPS协议，自动提取主机、端口和路径
3. **错误处理**：包含完整的异常捕获和错误信息输出
4. **响应解析**：支持多种响应格式（content字段、reasoning_content字段、text字段）
5. **性能统计**：精确统计token消耗、请求时间和token速度
6. **流式响应**：支持Server-Sent Events（SSE），实现实时输出
7. **会话管理**：自动维护聊天历史记录，支持上下文窗口管理
8. **终端交互**：实现命令行界面的交互式聊天功能

## 学习建议

### practice01学习路径
1. 首先运行 `python practice01/llm_client.py` 观察输出结果
2. 修改.env文件中的配置参数，观察不同参数对结果的影响
3. 修改请求提示词，观察LLM的响应变化
4. 尝试修改代码，添加更多的功能和统计指标

### practice02学习路径
1. 运行 `python practice02/chat_client.py` 启动交互式聊天
2. 输入不同的消息，体验流式输出效果
3. 尝试多轮对话，观察历史记录如何影响AI的响应
4. 修改代码中的历史记录长度限制，观察不同设置的效果
5. 尝试实现对话历史的保存和加载功能

## 注意事项

1. 确保LLM服务正在运行，并且可以通过配置的BASE_URL访问
2. 确保API_KEY正确，否则会收到认证错误
3. 网络请求可能需要一定时间，请耐心等待响应
4. 如果遇到超时错误，请检查LLM服务是否正常运行
